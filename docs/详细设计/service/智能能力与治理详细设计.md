## 2. 模块职责

### 2.1 `ai-orchestration-service`

- 问答编排、模型配置读取与写入。
- 工具调用、上下文聚合、结果治理。

### 2.2 `ocr-service`

- 验证码识别与结果标准化。
- OCR 任务记录、结果追踪、错误返回。

### 2.3 `capability-registry-service`

- 跨端能力版本矩阵维护。
- 能力开放状态、降级策略、工具可见性控制。

## 3. 领域边界

- AI 只负责编排与增强，不直接持有交易主链路状态。
- OCR 只负责识别，不负责券商登录主状态机。
- 能力注册中心只维护能力元数据，不维护实际业务状态。

## 4. 核心领域模型

| 对象 | 说明 |
| --- | --- |
| `ModelConfig` | 模型配置 |
| `Conversation` | 问答会话 |
| `ToolCall` | 工具调用记录 |
| `OCRJob` | OCR 任务 |
| `CapabilityDefinition` | 能力定义 |
| `CapabilityVersion` | 能力开放版本与降级策略 |

## 5. 核心业务流程

### 5.1 AI 问答与工具调用

1. 接收用户问题。
2. 查询 `capability-registry-service` 获取允许使用的工具和版本。
3. 聚合行情、资讯、策略、交易上下文。
4. 调用模型并执行工具链。
5. 输出结果并发布工具调用审计事件。

### 5.2 OCR 验证码识别

1. 接收图片或任务请求。
2. 调用 OCR 供应商识别。
3. 返回标准化文本、错误码、置信度。
4. 发布 `ocr.job_finished` 事件。

### 5.3 能力版本治理

1. 维护能力 ID、开放版本、跨端可见性、降级策略。
2. 为网关和 AI 工具调用提供能力判断接口。
3. 对未开放能力返回统一降级说明。

## 8. 接口设计

### 8.1 HTTP 入口

- `/api/v1/ai/chat`
- `/api/v1/ai/models`
- `/api/v1/ocr/jobs`
- `/api/v1/capabilities`

#### 8.1.1 `POST /api/v1/ai/chat`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `conversation_id` | string | 否 | 不传则创建新会话 |
| `question` | string | 是 | 用户问题 |
| `context_scope` | array | 否 | 如 `market`、`news`、`strategy`、`trade` |
| `tool_preference` | array | 否 | 允许优先使用的工具 |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `conversation_id` | string | 会话 ID |
| `answer` | string | 最终回答 |
| `tool_calls` | array | 工具调用摘要 |
| `model_name` | string | 使用模型 |

#### 8.1.2 `GET /api/v1/ai/models`

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `items` | array | 模型列表 |
| `items[].model_id` | string | 模型 ID |
| `items[].provider_code` | string | 提供方 |
| `items[].enabled` | boolean | 是否启用 |

#### 8.1.3 `POST /api/v1/ocr/jobs`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `image_ref` | string | 是 | 图片引用或上传返回 ID |
| `scene` | string | 是 | 如 `broker_captcha` |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `job_id` | string | OCR 任务 ID |
| `status` | string | 任务状态 |
| `text` | string | 识别结果，可空 |
| `confidence` | number | 置信度 |
| `reason_code` | string | 失败原因码，可空 |

#### 8.1.4 `GET /api/v1/capabilities`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `client_type` | string | 否 | `web`、`cli`、`ai` |
| `version` | string | 否 | 客户端版本 |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `items` | array | 能力列表 |
| `items[].capability_id` | string | 能力 ID |
| `items[].enabled` | boolean | 是否启用 |
| `items[].open_version` | string | 开放版本 |
| `items[].degrade_strategy` | string | 降级策略 |

## 9. 实施要点

- AI 工具调用必须经过工具白名单和资源权限校验。
- AI 可以提供建议和解释，但不能绕过交易风控直接修改交易事实。
- 能力注册中心必须成为前端、CLI、AI 的统一版本口径来源。
