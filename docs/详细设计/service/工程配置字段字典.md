# TradingClaw 工程配置字段字典

## 1. 文档定位

- 本文档是 `配置与密钥管理详细设计.md` 的字段级展开，定义工程配置命名空间、字段说明、类型、默认值、必填规则和热更新策略。
- 本文档只覆盖平台运行配置，不覆盖业务接口字段、领域事件 payload 或业务主数据字段。
- 任何新增运行配置字段都必须先登记本字典，再进入代码实现和环境发布。

- 总体总览：`docs/详细设计/service/后端详细设计.md`

## 2. 字段约定

### 2.1 通用列说明

| 列 | 说明 |
| --- | --- |
| `字段` | YAML 路径 |
| `类型` | 逻辑类型 |
| `必填` | `是` / `否` / `条件必填` |
| `默认值` | 若无则显式写 `无` |
| `热更新` | `允许` / `禁止` / `受限` |
| `说明` | 语义与约束 |

### 2.2 通用规则

- 生产环境默认禁止未知字段。
- `*_ref` 表示引用型字段，内容通常为 `vault://`、`file://` 或对象存储引用，不直接保存实体内容。
- 同一字段若在不同角色下语义不同，应拆为角色级子命名空间，不允许复用同名键承载不同含义。

## 21. 字段治理规则

- 新增字段必须明确 owner、默认值、热更新属性和环境适用范围。
- 删除字段必须先经历弃用阶段，至少跨一个发布窗口保留兼容读取。
- 同一字段的默认值不得在不同环境随意变化，若确需变化必须通过环境覆盖显式记录。

## 22. 结论

TradingClaw 的工程配置必须使用统一字段字典管理。平台实现、配置渲染器、Pydantic Settings、Helm values、运行诊断工具和发布守门都应以本字典为单一口径来源，禁止出现“代码一套、配置文件一套、部署模板一套”的漂移。
