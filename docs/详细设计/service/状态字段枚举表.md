# TradingClaw 状态字段枚举表

## 1. 文档定位

- 本文档统一 `docs/详细设计/service/` 下状态字段与专用枚举字段的取值口径。
- 目标是让前端、CLI、HTTP 接口、gRPC 接口、事件、工作流和各领域服务对同一状态值保持一致理解。
- 本文档只定义标准枚举，不定义通道原始状态；外部系统原始值必须通过 `raw_status` 等专用字段保留。

## 2. 使用规则

- `status` 是资源自己的标准化业务状态，不是全局通用枚举；必须结合资源类型解释。
- `auth_session_status`、`binding_status`、`trading_session_status`、`adapter_session_status`、`runtime_status`、`data_status`、`severity`、`decision` 等专用字段必须使用对应专用枚举，不能混用 `status`。
- 外部通道原始状态必须先映射到领域标准状态，再映射为展示状态。
- 新增状态值时，必须同步更新状态机、事件文档、接口文档和对应模块详细设计。
- 除特别说明外，枚举值区分大小写；大部分业务状态使用全大写，平台与账户能力开放状态沿用小写值。

## 3. 字段总览

| 字段 | 适用对象 | 说明 |
| --- | --- | --- |
| `status` | 用户、订单、策略实例、风险事件、工作流、OCR、通知、AI 会话、AI 工具调用、审计记录 | 资源自己的标准化业务状态 |
| `auth_session_status` | 登录会话 | 身份域登录会话状态 |
| `binding_status` | 账户绑定 | 用户与账户绑定关系状态 |
| `account_capability_status` | 账户能力投影 | 账户能力可用状态 |
| `trading_session_status` | 统一交易会话 | 交易网关统一交易会话状态 |
| `adapter_session_status` | 适配侧会话 | 券商或交易所适配侧会话状态 |
| `runtime_status` | 策略运行时 | 策略执行上下文状态 |
| `data_status` | 行情、持仓、余额等读模型 | 数据新鲜度与可用性状态 |
| `time_in_force` | 订单 | 订单时效 |
| `session_scope` | 订单 | 交易时段范围 |
| `settlement_rule` | 交易规则 | 交收规则 |
| `severity` | 风险事件 | 风险等级 |
| `decision` | 风控裁决、复核结论 | 风控决策结果 |
| `capability_status` | 平台能力治理 | 平台能力开放状态 |

## 4. 身份与账户域

### 4.1 用户状态枚举

字段：`status`

适用对象：`User`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `ACTIVE` | 用户可正常使用系统 | 否 |
| `LOCKED` | 用户因安全或风控原因被锁定 | 否 |
| `DISABLED` | 用户被禁用 | 是 |

### 4.2 登录会话状态枚举

字段：`auth_session_status`

适用对象：`Session`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `CREATED` | 会话已创建但尚未完全激活 | 否 |
| `ACTIVE` | 会话有效，可用于鉴权 | 否 |
| `EXPIRED` | 会话自然过期 | 是 |
| `REVOKED` | 会话被主动登出或吊销 | 是 |
| `LOCKED` | 会话因安全风险被冻结 | 否 |

允许迁移：

- `CREATED -> ACTIVE`
- `ACTIVE -> EXPIRED`
- `ACTIVE -> REVOKED`
- `ACTIVE -> LOCKED`
- `LOCKED -> ACTIVE`
- `LOCKED -> REVOKED`

### 4.3 账户绑定状态枚举

字段：`binding_status`

适用对象：`AccountBinding`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `PENDING_VERIFY` | 等待绑定校验 | 否 |
| `BOUND` | 已绑定 | 否 |
| `UNBOUND` | 已解绑 | 是 |
| `DISABLED` | 被系统禁用 | 是 |

补充规则：

- `BOUND` 只表示账户归属关系已建立，不代表能力已验证或交易会话已可用。

### 4.4 账户能力状态枚举

字段：`account_capability_status`

适用对象：`AccountCapabilityProjection`

| 枚举值 | 说明 |
| --- | --- |
| `pending_verify` | 已绑定但尚未完成凭据或通道校验 |
| `enabled` | 凭据或通道能力已验证可用 |
| `disabled` | 能力不可用或校验失败 |
| `degraded` | 能力部分可用或受限 |

补充规则：

- `account_capability_status` 描述账户接入能力，不描述平台开放状态。
- 平台开放状态统一使用 `capability_status`，不得混用。

## 5. 交易域

### 5.1 统一交易会话状态枚举

字段：`trading_session_status`

适用对象：`TradingSession`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `UNINITIALIZED` | 尚未建立交易认证上下文 | 否 |
| `AUTHENTICATING` | 正在认证 | 否 |
| `AVAILABLE` | 可执行交易动作 | 否 |
| `DEGRADED` | 可读不可写或部分受限 | 否 |
| `EXPIRED` | 认证失效 | 否 |
| `FAILED` | 建立失败 | 否 |
| `CLOSED` | 显式关闭，不可恢复 | 是 |

允许迁移：

- `UNINITIALIZED -> AUTHENTICATING`
- `AUTHENTICATING -> AVAILABLE`
- `AUTHENTICATING -> FAILED`
- `AVAILABLE -> DEGRADED`
- `AVAILABLE -> EXPIRED`
- `AVAILABLE -> CLOSED`
- `DEGRADED -> AUTHENTICATING`
- `DEGRADED -> EXPIRED`
- `EXPIRED -> AUTHENTICATING`

补充规则：

- 只有 `AVAILABLE` 才允许真实交易写操作。
- `default_trading_session_id` 只允许引用 `AVAILABLE` 的统一交易会话。

### 5.2 适配侧会话状态枚举

字段：`adapter_session_status`

适用对象：券商或交易所适配侧会话

| 枚举值 | 说明 |
| --- | --- |
| `UNINITIALIZED` | 适配侧尚未建立会话 |
| `AUTHENTICATING` | 适配侧认证中 |
| `AVAILABLE` | 适配侧会话可用 |
| `DEGRADED` | 适配侧会话受限 |
| `EXPIRED` | 适配侧会话过期 |
| `FAILED` | 适配侧会话建立失败 |
| `CLOSED` | 适配侧会话已关闭 |

补充规则：

- `adapter_session_status` 只在适配域内部和适配查询接口中使用。
- 对外统一交易语义必须使用 `trading_session_status`。

### 5.3 统一订单状态枚举

字段：`status`

适用对象：`Order`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `CREATED` | 统一订单已写入主记录 | 否 |
| `PENDING_REVIEW` | 命中人工复核，等待审核结论 | 否 |
| `PENDING_SUBMIT` | 准备投递到通道 | 否 |
| `SUBMITTED` | 已发往通道，等待确认 | 否 |
| `ACCEPTED` | 通道已接受 | 否 |
| `PARTIALLY_FILLED` | 部分成交 | 否 |
| `FILLED` | 全部成交完成 | 是 |
| `CANCEL_REQUESTED` | 已发起撤单 | 否 |
| `CANCELLED` | 撤单成功 | 是 |
| `REJECTED` | 通道拒绝或复核拒绝 | 是 |
| `FAILED` | 系统失败或通道终态失败 | 是 |
| `EXPIRED` | 订单超时失效 | 是 |

补充规则：

- 命中人工复核时，订单进入 `PENDING_REVIEW`。
- 复核通过后，订单从 `PENDING_REVIEW` 进入 `PENDING_SUBMIT`。
- 复核拒绝或处置终止后，订单从 `PENDING_REVIEW` 进入 `REJECTED`。
- 通道或交易时段导致的超时失效可从 `SUBMITTED`、`ACCEPTED` 进入 `EXPIRED`。

### 5.4 订单时效枚举

字段：`time_in_force`

| 枚举值 | 说明 |
| --- | --- |
| `DAY` | 当日有效 |
| `GTC` | 撤销前有效 |
| `IOC` | 立即成交剩余取消 |

### 5.6 交收规则枚举

字段：`settlement_rule`

| 枚举值 | 说明 |
| --- | --- |
| `T_PLUS_1` | T+1 交收 |

### 5.7 工作流实例状态枚举

字段：`status`

适用对象：工作流实例

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `CREATED` | 工作流已创建 | 否 |
| `RUNNING` | 工作流执行中 | 否 |
| `WAITING_SIGNAL` | 等待外部信号或定时器 | 否 |

| `COMPENSATING` | 执行补偿中 | 否 |
| `SUCCEEDED` | 工作流成功结束 | 是 |

## 6. 策略域

### 6.1 策略实例状态枚举

字段：`status`

适用对象：`StrategyInstance`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `PENDING` | 待启动 | 否 |
| `RUNNING` | 运行中 | 否 |
| `STOPPED` | 已停止，可恢复 | 否 |
| `OFFLINED` | 已下线，对应删除语义 | 否 |
| `ARCHIVED` | 已归档，不可恢复运行 | 是 |

允许迁移：

- `PENDING -> RUNNING`
- `RUNNING -> STOPPED`
- `STOPPED -> RUNNING`
- `PENDING/RUNNING/STOPPED -> OFFLINED`
- `OFFLINED -> ARCHIVED`

### 6.2 策略运行时状态枚举

字段：`runtime_status`

适用对象：策略运行上下文

| 枚举值 | 说明 |
| --- | --- |
| `INIT` | 初始化中 |
| `READY` | 依赖加载完成，等待触发 |
| `EVALUATING` | 执行策略计算 |
| `PLANNING` | 生成交易计划 |
| `EXECUTING` | 执行交易计划 |
| `WAITING_EVENT` | 等待事件输入 |
| `RECOVERING` | 异常恢复中 |
| `PAUSED` | 被人工或系统暂停 |
| `TERMINATED` | 运行终止 |

补充规则：

- `PAUSED` 用于表达人工复核等待或显式暂停，不等价于策略实例 `STOPPED`。

## 10. 跨文档约束

- HTTP、gRPC、事件中的状态字段必须共用本文档中的标准值。
- 前端和 CLI 可以根据标准值派生展示文案，但不能自定义新的状态值替代标准值。
- 若需保留外部系统原始状态，必须额外使用 `raw_status` 等专用字段，不得覆盖标准化状态字段。
- `status` 同名但跨资源语义不同，落地实现时必须由资源类型绑定对应枚举，不得做全局混用。
