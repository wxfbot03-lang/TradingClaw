# TradingClaw 状态字段枚举表

## 1. 文档说明

- 本文档用于统一 `docs/详细设计/service/` 下各模块涉及的状态字段枚举口径。
- 目标是确保前端、CLI、网关、领域服务、事件、工作流对状态值理解一致。
- 本文档聚焦枚举值定义、状态语义、允许迁移和适用范围。

## 2. 使用规则

- `status` 仅表示当前对象的标准化业务状态。
- `auth_session_status`、`trading_session_status`、`runtime_status`、`binding_status` 等专用状态字段必须使用对应枚举，不能混用通用 `status` 值。
- 外部通道原始状态必须先映射到内部标准化状态，再映射给前端或 CLI。
- 新增状态值必须同步更新状态机、事件和接口文档。

## 3. 用户会话状态枚举

字段：`auth_session_status`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `CREATED` | 会话已创建但尚未完全激活 | 否 |
| `ACTIVE` | 会话有效，可用于鉴权 | 否 |
| `EXPIRED` | 会话自然过期 | 是 |
| `REVOKED` | 会话被主动登出或吊销 | 是 |
| `LOCKED` | 会话因安全风险被冻结 | 否 |

允许迁移：

- `CREATED -> ACTIVE`
- `ACTIVE -> EXPIRED`
- `ACTIVE -> REVOKED`
- `ACTIVE -> LOCKED`
- `LOCKED -> ACTIVE`
- `LOCKED -> REVOKED`

## 4. 交易会话状态枚举

字段：`trading_session_status`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `UNINITIALIZED` | 尚未建立交易认证上下文 | 否 |
| `AUTHENTICATING` | 正在认证 | 否 |
| `AVAILABLE` | 可执行交易动作 | 否 |
| `DEGRADED` | 可读不可写或部分受限 | 否 |
| `EXPIRED` | 认证失效 | 否 |
| `FAILED` | 建立失败 | 否 |
| `CLOSED` | 显式关闭，不可恢复 | 是 |

允许迁移：

- `UNINITIALIZED -> AUTHENTICATING`
- `AUTHENTICATING -> AVAILABLE`
- `AUTHENTICATING -> FAILED`
- `AVAILABLE -> DEGRADED`
- `AVAILABLE -> EXPIRED`
- `AVAILABLE -> CLOSED`
- `DEGRADED -> AUTHENTICATING`
- `DEGRADED -> EXPIRED`
- `EXPIRED -> AUTHENTICATING`

## 5. 账户绑定状态枚举

字段：`binding_status`

| 枚举值 | 说明 |
| --- | --- |
| `BOUND` | 已绑定 |
| `UNBOUND` | 已解绑 |
| `PENDING_VERIFY` | 等待校验 |
| `DISABLED` | 被系统禁用 |

补充规则：

## 6. 统一订单状态枚举

字段：`status`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `CREATED` | 统一订单已写入主记录 | 否 |
| `PENDING_REVIEW` | 命中人工复核，等待审核结论 | 否 |
| `PENDING_SUBMIT` | 准备投递到通道 | 否 |
| `SUBMITTED` | 已发往通道，等待确认 | 否 |
| `ACCEPTED` | 通道已接受 | 否 |
| `PARTIALLY_FILLED` | 部分成交 | 否 |
| `FILLED` | 全部成交完成 | 是 |
| `CANCEL_REQUESTED` | 已发起撤单 | 否 |
| `CANCELLED` | 撤单成功 | 是 |
| `REJECTED` | 通道拒绝 | 是 |
| `FAILED` | 系统失败或通道终态失败 | 是 |
| `EXPIRED` | 订单超时失效 | 是 |

补充规则：

- 命中人工复核时，订单进入 `PENDING_REVIEW`。
- 复核通过后，订单从 `PENDING_REVIEW` 进入 `PENDING_SUBMIT`。
- 复核拒绝或处置终止后，订单从 `PENDING_REVIEW` 进入 `REJECTED`。
- 通道或交易时段导致的超时失效可从 `SUBMITTED`、`ACCEPTED` 进入 `EXPIRED`。

## 7. 策略实例状态枚举

字段：`status`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `PENDING` | 待启动 | 否 |
| `RUNNING` | 运行中 | 否 |
| `STOPPED` | 已停止，可恢复 | 否 |
| `OFFLINED` | 已下线，对应删除语义 | 否 |
| `ARCHIVED` | 已归档，不可恢复运行 | 是 |

允许迁移：

- `PENDING -> RUNNING`
- `RUNNING -> STOPPED`
- `STOPPED -> RUNNING`
- `PENDING/RUNNING/STOPPED -> OFFLINED`
- `OFFLINED -> ARCHIVED`

## 27. 取值约束

- HTTP、gRPC、事件中的状态字段必须共用同一标准值。
- 前端和 CLI 不能自定义展示状态来替代标准状态值。
- 若为兼容外部通道需要保留原始状态，应额外使用 `raw_status` 字段，不得覆盖标准化 `status`。
