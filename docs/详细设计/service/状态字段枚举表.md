# TradingClaw 状态字段枚举表

## 1. 文档说明

- 本文档用于统一 `docs/详细设计/service/` 下各模块涉及的状态字段枚举口径。
- 目标是确保前端、CLI、网关、领域服务、事件、工作流对状态值理解一致。
- 本文档聚焦枚举值定义、状态语义、允许迁移和适用范围。

## 2. 使用规则

- `status` 仅表示当前对象的标准化业务状态。
- `session_status`、`runtime_status`、`binding_status` 等专用状态字段必须使用对应枚举，不能混用通用 `status` 值。
- 外部通道原始状态必须先映射到内部标准化状态，再映射给前端或 CLI。
- 新增状态值必须同步更新状态机、事件和接口文档。

## 3. 用户会话状态枚举

字段：`session_status`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `CREATED` | 会话已创建但尚未完全激活 | 否 |
| `ACTIVE` | 会话有效，可用于鉴权 | 否 |
| `EXPIRED` | 会话自然过期 | 是 |
| `REVOKED` | 会话被主动登出或吊销 | 是 |
| `LOCKED` | 会话因安全风险被冻结 | 否 |

允许迁移：

- `CREATED -> ACTIVE`
- `ACTIVE -> EXPIRED`
- `ACTIVE -> REVOKED`
- `ACTIVE -> LOCKED`
- `LOCKED -> ACTIVE`
- `LOCKED -> REVOKED`

## 7. 策略实例状态枚举

字段：`status`

| 枚举值 | 说明 | 终态 |
| --- | --- | --- |
| `PENDING` | 待启动 | 否 |
| `RUNNING` | 运行中 | 否 |
| `STOPPED` | 已停止，可恢复 | 否 |
| `OFFLINED` | 已下线，对应删除语义 | 否 |
| `ARCHIVED` | 已归档，不可恢复运行 | 是 |

允许迁移：

- `PENDING -> RUNNING`
- `RUNNING -> STOPPED`
- `STOPPED -> RUNNING`
- `PENDING/RUNNING/STOPPED -> OFFLINED`
- `OFFLINED -> ARCHIVED`

## 16. 取值约束

- HTTP、gRPC、事件中的状态字段必须共用同一标准值。
- 前端和 CLI 不能自定义展示状态来替代标准状态值。
- 若为兼容外部通道需要保留原始状态，应额外使用 `raw_status` 字段，不得覆盖标准化 `status`。
