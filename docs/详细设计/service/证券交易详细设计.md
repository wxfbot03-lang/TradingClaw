# TradingClaw 证券交易详细设计

## 1. 范围说明

- 本文档覆盖 `securities-trading-service`。
- 对应需求主要包括 `SEC-001` ~ `SEC-005`。
- 本模块负责券商通道适配，不负责统一交易模型对外暴露。

## 2. 模块职责

- 券商登录、验证码获取、会话保持。
- 证券账户资产、持仓、委托、成交查询。
- 股票/基金下单与撤单的券商协议执行。
- 将原始券商回报标准化后回传交易网关。

## 3. 服务边界

- 对上游只暴露通道适配能力，不直接面向客户端。
- 统一订单状态、统一错误码、统一会话抽象由交易网关维护。
- 策略模块和客户端不得直接调用券商 SDK。

## 4. 核心模型

| 对象 | 说明 |
| --- | --- |
| `BrokerLoginTask` | 券商登录任务 |
| `CaptchaJob` | 验证码任务 |
| `BrokerSession` | 券商会话上下文 |
| `SecuritiesAssetSnapshot` | 证券资产快照 |
| `SecuritiesPositionSnapshot` | 证券持仓快照 |
| `SecuritiesOrderRecord` | 券商订单记录 |
| `SecuritiesTradeRecord` | 券商成交记录 |

## 5. 核心业务流程

### 5.1 券商登录

1. 触发 `BrokerLoginWorkflow`。
2. 获取验证码或执行免验证码认证。
3. 调用券商登录接口。
4. 建立 `BrokerSession` 并回写统一交易会话可用状态。
5. 登录失败时记录原因码并支持重试或人工介入。

### 5.2 证券下单

1. 接收来自交易网关的标准化下单命令。
2. 映射为券商协议字段。
3. 调用券商委托接口。
4. 返回原始结果并投递标准化回报事件。

### 5.3 券商回报同步

1. 接收主动回调或轮询结果。
2. 将原始状态标准化为回报对象。
3. 发布 `execution.report_received` 事件。
4. 归档原始报文到对象存储。

核心事件：

- `execution.report_received`
- `trading_session.expired`
- `notification.requested`（登录异常或风险提示时）

## 8. 接口设计

### 8.1 gRPC 服务

- `BrokerLoginService`
- `SecuritiesOrderAdapterService`
- `SecuritiesAssetService`
- `SecuritiesQueryService`

#### 8.1.1 `BrokerLoginService.Login`

请求字段：`account_id`、`broker_code`、`credential_ref`、`captcha_text`

响应字段：`broker_session_id`、`session_status`、`reason_code`

#### 8.1.2 `SecuritiesOrderAdapterService.PlaceOrder`

请求字段：`account_id`、`channel_order_request`

响应字段：`channel_order_id`、`raw_status`、`accepted`

#### 8.1.3 `SecuritiesAssetService.GetAssets`

请求字段：`account_id`

响应字段：`asset_summary`、`positions`

#### 8.1.4 `SecuritiesQueryService.GetOrderStatus`

请求字段：`account_id`、`channel_order_id`

响应字段：`raw_status`、`normalized_report`

### 8.2 工作流活动

- `request_broker_captcha`
- `submit_broker_login`
- `refresh_broker_session`
- `query_broker_order_status`

活动输入输出建议：

| 活动 | 输入 | 输出 |
| --- | --- | --- |
| `request_broker_captcha` | `account_id`、`broker_code` | `captcha_job_id`、`image_ref` |
| `submit_broker_login` | `account_id`、`credential_ref`、`captcha_text` | `broker_session_id`、`session_status` |
| `refresh_broker_session` | `broker_session_id` | `session_status`、`expired_at` |
| `query_broker_order_status` | `account_id`、`channel_order_id` | `raw_status`、`normalized_report` |

## 9. 实施要点

- 所有券商报文必须保留原始归档。
- 所有券商状态映射必须经过统一状态表，不允许在多个位置散落转换逻辑。
- 券商登录失败原因必须结构化记录，供前端和 CLI 返回可读提示。
