# TradingClaw 数字资产交易详细设计

## 1. 文档定位

- 本文档覆盖 `crypto-trading-service`，定义交易所适配域的凭据校验、会话管理、原始下单、原始查询和扩展行情能力设计。
- 本模块负责交易所 API 适配，不直接承载统一交易语义。
- 统一交易会话、统一订单状态、统一持仓余额和统一风控接入均由交易网关负责，本模块只输出适配侧事实。

## 2. 模块定位

- 负责 API Key / Secret 等交易所凭据的有效性校验。
- 负责数字资产余额、持仓、订单、成交流水和资金费率等原始查询。
- 负责现货、合约等订单的创建、修改、撤销和状态同步。
- 为交易网关和策略运行时提供受限的底层网格挂钩能力，但不承担完整网格策略状态机。

## 3. 领域边界

- 原始查询接口仅供 `trade-gateway-service`、工作流活动和运维排障使用，不得被客户端、策略运行时或 AI 直接依赖。
- 所有真实挂单、改单、撤单动作都必须先进入统一交易网关。
- `strategy-runtime-service` 不得直接调用 `crypto-trading-service` 发起真实交易写操作。
- 本模块不直接写入账户域能力投影表，不发布账户域最终事件或统一交易会话事件，只发布适配侧事实。

## 5. 交易所适配主线

### 5.1 凭据校验与会话主线

流程如下：

1. 接收账户凭据引用。
2. 从密钥管理系统获取实际凭据。
3. 调用交易所鉴权接口验证有效性。
4. 建立或刷新 `ExchangeSession`，并记录 `adapter_session_status`。
5. 发布 `exchange_credential.validated` 事件，并将能力矩阵供账户域投影使用。

约束：

- 所有凭据读取必须通过密钥管理系统完成。
- `adapter_session_status` 只表达适配侧连接状态，对外统一交易语义仍由交易网关负责。

### 5.2 普通订单执行主线

流程如下：

1. 接收交易网关下发的标准化订单。
2. 转换为交易所下单参数，并结合能力矩阵补齐只减仓、杠杆、保证金模式等约束。
3. 提交下单、改单或撤单请求。
4. 记录原始请求与原始响应引用。
5. 发布 `execution.report_received` 事件，供交易网关继续统一状态推进。

约束：

- 本模块返回的是适配侧原始结果和适配标准化结果，不是统一订单事实。
- 必须提供“交易所原始状态 -> 统一订单状态 -> 展示状态”的映射表，并区分现货、合约、只减仓等差异。

### 5.3 资金费率与扩展查询主线

流程如下：

1. 接收查询请求或定时拉取任务。
2. 访问交易所公共接口。
3. 标准化资金费率或扩展行情结果。
4. 写入快照或分析存储，供策略、风控和研究能力使用。

### 5.4 网格底层挂钩主线

流程如下：

1. 接收来自 `trade-gateway-service` 的受限网格动作请求。
2. 创建底层挂单、改单或调整任务。
3. 返回执行结果与子单关系。
4. 仅保存适配层执行记录，不在本模块维护完整网格策略状态机。

## 6. 数据设计

核心表：

- `exchange_sessions`
- `crypto_balances_snapshots`
- `crypto_positions_snapshots`
- `crypto_order_records`
- `funding_rate_snapshots`
- `grid_adapter_tasks`

设计要点：

- 交易所会话、订单回报、资金费率快照建议落 MySQL，保证适配器问题排查和历史追踪能力。
- `crypto_balances_snapshots`、`crypto_positions_snapshots` 用于读模型聚合，Redis 仅缓存最新展示结果。
- 网格适配任务只保存适配层执行记录，不承担完整策略事实。
- 默认值建议：`exchange_sessions.adapter_session_status = AUTHENTICATING`，`grid_adapter_tasks.status = CREATED`。
- 交易所会话、订单记录和资金费率快照默认不物理删除；读模型快照可按保留期滚动清理。

## 7. 事件设计

核心事件：

- `execution.report_received`
- `exchange_session.updated`
- `exchange_credential.validated`

事件约束：

- 本模块只发布适配侧事实，不发布统一交易会话事件或统一订单事实事件。
- 能力事件用于账户域投影刷新，不代表账户域最终主权事实。

## 8. 接口设计

### 8.1 gRPC 服务

- `CryptoCredentialService`
- `CryptoOrderAdapterService`
- `CryptoBalanceAdapterService`
- `CryptoFundingRateService`

#### 8.1.1 `CryptoCredentialService.Validate`

请求字段：`account_id`、`credential_ref`

响应字段：`valid`、`capabilities`、`account_capability_status`、`reason_code`

#### 8.1.2 `CryptoOrderAdapterService.PlaceOrder`

请求字段：`account_id`、`channel_order_request`

响应字段：`channel_order_id`、`raw_status`、`accepted`

#### 8.1.3 `CryptoBalanceAdapterService.GetBalancesRaw`

请求字段：`account_id`

响应字段：`balances`、`positions`

语义约束：仅供 `trade-gateway-service`、工作流活动和运维排障使用，不作为统一余额或持仓事实接口。

#### 8.1.4 `CryptoFundingRateService.GetFundingRates`

请求字段：`symbols`

响应字段：`items[].symbol`、`items[].funding_rate`、`items[].timestamp`

## 9. 实施要点

- 交易所能力差异较大，必须通过适配器能力矩阵暴露，不允许向上游承诺统一支持所有功能。
- 网格相关能力仅提供底层挂钩，不在本模块实现完整策略闭环。
- 必须提供首批交易所能力矩阵样例，至少覆盖杠杆范围、保证金模式、只减仓、订单时效和资金费率接口。
- 必须提供交易所错误码到统一错误码的映射细则，通道透传信息只用于排障，不直接暴露为统一业务状态。

## 10. 数字资产接入 API 能力清单

本节只整理交易所侧需要提供的 API 功能能力，不约束其参数和返回值格式；具体接口定义、字段名、错误码和报文结构由交易所提供后再做适配映射。

### 10.1 MVP 必需能力

| 类别 | API 功能 | 用途 | 说明 |
| --- | --- | --- | --- |
| 认证接入 | API 凭据校验 | 校验 API Key / Secret / Passphrase 是否有效 | 用于账户接入校验和能力投影 |
| 认证接入 | 会话建立或连接校验 | 建立或确认当前可用连接态 | 即使交易所是无状态签名接口，也需要有等价可用性校验能力 |
| 账户查询 | 余额查询 | 获取总余额、可用余额、冻结余额 | 用于交易前校验和统一余额读模型回补 |
| 账户查询 | 持仓查询 | 获取现货持仓或合约持仓 | 用于策略、交易和风控校验 |
| 账户查询 | 订单查询 | 查询单笔订单当前状态 | 用于查单、补偿和排障 |
| 账户查询 | 成交查询 | 查询订单成交明细 | 用于成交回补和对账 |
| 交易写入 | 现货下单 | 提交现货买卖订单 | 最低接入闭环必需 |
| 交易写入 | 撤单 | 对未终结订单发起撤销 | 需能返回撤单受理结果 |
| 回报同步 | 回报轮询查询 | 定时拉取订单、成交、账户增量变化 | 作为无推送场景主方案，或推送失败时兜底 |
| 交易规则 | 交易对列表查询 | 获取支持交易的 symbol 列表 | 用于市场适配和能力矩阵 |
| 交易规则 | 交易规则查询 | 获取最小下单数量、价格精度、数量精度等 | 用于构建 `TradingRuleProfile` |
| 交易规则 | 错误码说明 | 提供交易所错误码语义 | 用于映射统一错误码和可读提示 |
| 稳定性支持 | 限频/限流规则说明 | 明确接口调用频控限制 | 用于适配层限流和重试策略 |

### 10.2 建议增强能力

| 类别 | API 功能 | 用途 | 说明 |
| --- | --- | --- | --- |
| 认证接入 | 会话刷新/签名有效性校验 | 保持连接可用或提前发现凭据失效 | 用于降低交易前失败概率 |
| 账户查询 | 账户基础信息查询 | 获取账户类型、子账户信息、交易权限等 | 用于能力矩阵和账户展示 |
| 账户查询 | 批量订单查询 | 批量查询多笔订单状态 | 用于回补效率优化 |
| 账户查询 | 历史订单查询 | 获取历史订单 | 用于审计和恢复 |
| 账户查询 | 历史成交查询 | 获取历史成交 | 用于审计、对账和恢复 |
| 交易写入 | 改单 | 修改价格、数量等订单属性 | 若交易所支持改单可降低撤单重挂成本 |
| 交易写入 | 批量撤单 | 批量撤销多个订单 | 用于异常补偿和策略提效 |
| 回报同步 | 订单回报推送 | 接收订单受理、拒绝、撤单等变化 | 若支持推送，可降低轮询压力 |
| 回报同步 | 成交回报推送 | 接收成交、部分成交等变化 | 若支持推送，可更快刷新统一交易事实 |
| 回报同步 | 账户变动推送 | 接收余额、持仓、保证金变化 | 用于快速刷新读模型 |
| 交易规则 | 订单时效支持查询 | 获取支持的 `time_in_force` 类型 | 如 `GTC`、`IOC`、`FOK` |
| 稳定性支持 | 幂等或客户端订单号支持 | 识别重复下单或支持客户端订单号 | 若交易所支持可提升幂等保障 |
| 稳定性支持 | 可用性与维护窗口通知能力 | 感知停机、维护、交易暂停 | 可通过接口、状态页或公告机制提供 |

### 10.3 条件必需或可选能力

| 类别 | API 功能 | 用途 | 说明 |
| --- | --- | --- | --- |
| 交易写入 | 合约/衍生品下单 | 提交合约、永续等订单 | 仅在目标交易所支持并且业务范围需要时必需 |
| 交易规则 | 杠杆与保证金规则查询 | 获取杠杆范围、保证金模式、维持保证金要求 | 衍生品交易场景需要 |
| 交易规则 | 只减仓能力查询 | 获取 `reduce_only` 是否支持 | 衍生品交易场景需要 |
| 交易规则 | 资金费率查询 | 获取资金费率和生效时间 | 永续或合约场景建议提供 |
| 账户查询 | 资金划转记录查询 | 获取主账户与子账户资金划转记录 | 仅在账户体系复杂或运营场景需要时接入 |
| 账户查询 | 子账户列表查询 | 获取子账户与权限结构 | 仅在交易所提供子账户能力时需要 |

补充说明：

- 最低可接入闭环通常需要：凭据校验、余额查询、持仓查询、现货下单、撤单、订单查询、成交查询或成交回补能力。
- 若业务范围包含合约、永续或杠杆交易，则还需要补充杠杆、保证金、只减仓和资金费率相关能力。
- 若交易所不支持主动回报推送，则必须提供可靠的订单/成交/账户变化轮询查询能力。
- 若交易所不提供交易规则查询接口，则至少需要以文档或静态配置方式提供规则清单，便于我方维护适配规则映射。
