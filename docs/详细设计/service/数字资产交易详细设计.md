# TradingClaw 数字资产交易详细设计

## 1. 范围说明

- 本文档覆盖 `crypto-trading-service`。
- 对应需求主要包括 `CRY-001` ~ `CRY-004`。
- 本模块负责交易所 API 适配，不直接承载统一交易语义。

## 2. 模块职责

- API Key / Secret 有效性校验。
- 数字资产余额、持仓、订单、成交流水、资金费率查询。
- 普通订单创建、修改、撤销、状态同步。
- 为后续网格策略提供底层挂单、详情、子单、收益提取挂钩能力。

## 4. 核心业务流程

### 4.1 密钥校验

1. 接收账户凭据引用。
2. 从密钥管理系统获取实际凭据。
3. 调用交易所鉴权接口验证有效性。
4. 结果回写账户能力矩阵并返回校验结果。

### 4.2 普通订单执行

1. 接收交易网关下发的统一订单。
2. 转换为交易所下单参数。
3. 提交订单或撤单请求。
4. 发布标准化 `execution.report_received` 事件。

### 4.3 资金费率与行情扩展查询

1. 接收查询请求或定时拉取任务。
2. 访问交易所公共接口。
3. 标准化并写入快照或分析存储。

### 4.4 网格底层挂钩

1. 接收来自策略运行时的网格动作请求。
2. 创建底层挂单或调整任务。
3. 返回执行结果和子单关系。
4. 不直接承载完整网格策略状态机。

## 7. 接口设计

### 7.1 gRPC 服务

- `CryptoCredentialService`
- `CryptoOrderAdapterService`
- `CryptoBalanceService`
- `CryptoFundingRateService`

#### 7.1.1 `CryptoCredentialService.Validate`

请求字段：`account_id`、`credential_ref`

响应字段：`valid`、`capabilities`、`reason_code`

#### 7.1.2 `CryptoOrderAdapterService.PlaceOrder`

请求字段：`account_id`、`channel_order_request`

响应字段：`channel_order_id`、`raw_status`、`accepted`

#### 7.1.3 `CryptoBalanceService.GetBalances`

请求字段：`account_id`

响应字段：`balances`、`positions`

#### 7.1.4 `CryptoFundingRateService.GetFundingRates`

请求字段：`symbols`

响应字段：`items[].symbol`、`items[].funding_rate`、`items[].timestamp`

### 7.2 工作流活动

- `validate_exchange_credentials`
- `refresh_exchange_session`
- `query_exchange_order_status`

活动输入输出建议：

| 活动 | 输入 | 输出 |
| --- | --- | --- |
| `validate_exchange_credentials` | `account_id`、`credential_ref` | `valid`、`capabilities` |
| `refresh_exchange_session` | `account_id`、`session_ref` | `session_status`、`expired_at` |
| `query_exchange_order_status` | `account_id`、`channel_order_id` | `raw_status`、`normalized_report` |

## 8. 实施要点

- 交易所能力差异较大，必须通过适配器能力矩阵暴露，不允许向上游承诺统一支持所有功能。
- 网格相关能力仅提供底层挂钩，不在本模块实现完整策略闭环。
- 所有凭据读取必须通过密钥管理系统完成。
