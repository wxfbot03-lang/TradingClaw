## 2. 模块职责

### 2.1 `market-service`

- 股票、基金、数字资产实时行情查询。
- 历史行情、K 线、均线等指标供给。
- 行情标准化、缓存回退、缺口识别与增量补齐。

### 2.2 `news-service`

- 多源资讯抓取、聚合、去重、分页查询。
- 资讯影响力评分与排序。

## 5. 核心业务流程

### 5.1 实时行情查询

1. 接收行情查询请求。
2. 优先从缓存读取热点数据。
3. 未命中时访问外部行情源。
4. 标准化后写回缓存并返回。
5. 外部依赖失败时回退可用缓存并标记数据状态。

### 5.2 历史行情补齐

1. 接收区间查询或离线回填任务。
2. 检查已有区间和缺口。
3. 通过 `HistoricalBackfillWorkflow` 补齐缺口。
4. 完成去重、排序、标准化后写入时序库。
5. 发布 `historical.backfilled` 事件。

### 5.3 指标计算

1. 读取标准化 K 线序列。
2. 根据指标类型和窗口进行计算。
3. 样本不足时返回不可计算状态。
4. 计算结果进入分析库并可发布 `indicator.calculated` 事件。

### 5.4 资讯聚合与评分

1. 从多个资讯源抓取原始新闻。
2. 标准化、清洗并建立去重指纹。
3. 入库标准资讯实体。
4. 执行影响力评分。
5. 写入检索索引并发布 `news.ingested`、`news.scored` 事件。

## 8. 接口设计

### 8.1 HTTP 入口

- `/api/v1/market/quotes`
- `/api/v1/market/history`
- `/api/v1/market/indicators`
- `/api/v1/news/items`
- `/api/v1/news/scores`

#### 8.1.1 `GET /api/v1/market/quotes`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `symbols` | string | 是 | 多个标的以逗号分隔 |
| `market` | string | 否 | 市场代码 |
| `with_depth` | boolean | 否 | 是否返回五档盘口 |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `quotes` | array | 行情列表 |
| `quotes[].symbol` | string | 标的代码 |
| `quotes[].price` | number | 最新价 |
| `quotes[].timestamp` | string | 数据时间 |
| `quotes[].source` | string | 数据源 |
| `quotes[].depth` | array | 五档盘口，可选 |
| `data_status` | string | `fresh` 或 `cache_fallback` |

#### 8.1.2 `GET /api/v1/market/history`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `symbol` | string | 是 | 标的代码 |
| `market` | string | 否 | 市场代码 |
| `period` | string | 是 | `1m`、`5m`、`1d` 等 |
| `start_at` | string | 否 | 起始时间 |
| `end_at` | string | 否 | 结束时间 |
| `limit` | integer | 否 | 返回条数上限 |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `symbol` | string | 标的代码 |
| `period` | string | 周期 |
| `items` | array | K 线列表 |
| `items[].open` | number | 开盘价 |
| `items[].high` | number | 最高价 |
| `items[].low` | number | 最低价 |
| `items[].close` | number | 收盘价 |
| `items[].volume` | number | 成交量 |
| `items[].bar_time` | string | K 线时间 |
| `data_status` | string | 数据状态 |

#### 8.1.3 `GET /api/v1/market/indicators`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `symbol` | string | 是 | 标的代码 |
| `indicator_type` | string | 是 | 如 `ma` |
| `window` | integer | 是 | 指标窗口 |
| `period` | string | 是 | 周期 |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `indicator_type` | string | 指标类型 |
| `window` | integer | 窗口 |
| `calculable` | boolean | 是否可计算 |
| `reason` | string | 不可计算原因 |
| `series` | array | 指标序列 |

#### 8.1.4 `GET /api/v1/news/items`

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `keyword` | string | 否 | 关键字 |
| `source_id` | string | 否 | 资讯源 |
| `page` | integer | 否 | 页码 |
| `page_size` | integer | 否 | 每页大小 |

返回体 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `items` | array | 资讯列表 |
| `items[].news_id` | string | 资讯 ID |
| `items[].title` | string | 标题 |
| `items[].published_at` | string | 发布时间 |
| `items[].source_id` | string | 来源 |

#### 8.1.5 `GET /api/v1/news/scores`

查询参数：

| 参数 | 类型 | 必填 | 说明 |

## 9. 依赖与实施顺序

- 本模块建议第二批实现。
- 是策略系统、AI、交易查询的重要基础数据依赖。
- 行情标准化口径必须在本模块统一，不允许策略和交易模块各自做二次定义。
