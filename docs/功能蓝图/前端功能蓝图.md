# TradingClaw 前端功能需求文档

## 1. 文档目的
1. 定义前端应用必须具备的完整用户功能，供团队独立实现。
2. 明确页面行为、交互流程、状态管理规则与验收标准。
3. 作为产品、设计、前端、测试协作的统一基线。

## 2. 产品定位与使用对象
### 2.1 产品定位
1. 提供投资业务的一站式前端工作台，覆盖策略管理、资产查询、交易操作、资讯查看与智能分析。
2. 兼容桌面端和移动端，保障主要业务在不同终端的可用性与一致性。

### 2.2 使用对象
1. 普通用户：完成注册、登录、查看资讯、使用基础功能。
2. 策略用户：创建、管理、查看交易机器人和策略结果。
3. 交易用户：管理账户连接并执行交易相关操作。

## 3. 全局功能要求
### 3.0 前端分层模型
1. 前端能力分为页面层与前端服务层两部分。
2. 页面层负责界面渲染、交互流程、状态展示与用户操作反馈。
3. 前端服务层负责配置管理、文档内容管理、前端业务聚合与对外服务接口。
4. 页面层必须通过前端服务层获取页面配置与文档数据，不直接维护源数据。
5. 页面展示文案必须由前端服务层提供，页面层不得写死业务文案。

### 3.1 应用壳与导航
1. 提供统一主框架，包含头部、侧边导航、主内容区。
2. 提供桌面侧栏导航与移动端抽屉导航两套交互模式。
3. 提供核心业务页面入口，保证页面间跳转一致。
4. 提供全局退出登录能力，退出后清理会话状态并跳转到认证页。

### 3.2 会话与权限
1. 提供登录态持久化，保存访问凭证、用户会话标识、用户信息等关键状态。
2. 提供路由访问控制，未登录用户仅可访问公开页面。
3. 提供登录后身份恢复能力，刷新页面后可继续工作。
4. 任何依赖会话的页面在会话失效时必须引导用户重新登录。

### 3.3 主题与响应式
1. 提供深色与浅色主题切换能力。
2. 支持跟随系统主题自动切换。
3. 主要页面支持移动端自适应布局。
4. 视觉样式需保持组件风格统一，保证可读性和可操作性。

### 3.4 配置驱动页面
1. 提供全局配置读取能力，驱动标题、按钮文案、表头、提示语等页面文案。
2. 提供页面级配置读取能力，支持不同业务页独立配置。
3. 页面配置由前端服务统一维护，页面层只消费配置结果。
4. 配置获取失败时必须有默认兜底文案，保证页面可用。
5. 提供配置缓存能力，减少重复请求并提升加载速度。
6. 默认兜底文案同样由前端服务层统一维护，不允许页面内写死业务文案。

## 4. 页面功能需求

### 4.1 首页与文档中心
#### 4.1.1 首页展示
1. 展示首页核心介绍信息与关键入口。
2. 展示页脚信息与品牌说明。
3. 提供进入用户认证页面的快捷入口。

#### 4.1.2 文档目录与阅读
1. 展示文档目录列表并支持点击切换。
2. 首次进入时默认打开首个文档。
3. 文档正文支持富文本渲染与格式化展示。
4. 移动端支持菜单展开收起与阅读区切换。
5. 文档加载中、加载失败、空数据需有明确状态提示。
6. 文档内容源由前端服务统一维护与发布，页面层只负责展示。

### 4.2 用户认证页面
#### 4.2.1 注册流程
1. 提供注册表单并进行前端输入校验。
2. 提供注册结果反馈，成功后引导用户进行登录。

#### 4.2.2 登录流程
1. 提供登录表单并进行字段校验。
2. 登录成功后保存访问凭证与用户会话信息。
3. 登录成功后跳转到默认业务页。
4. 登录失败时展示明确错误提示。

### 4.3 账户设置页面
#### 4.3.1 交易账户管理
1. 展示当前用户已绑定交易账户列表。
2. 提供新增交易账户入口，并通过授权登录流程完成账户接入。
3. 提供删除交易账户能力。
4. 账户变更后自动刷新列表状态。

#### 4.3.2 模型设置管理
1. 展示当前模型配置状态。
2. 提供模型密钥与自动登录等参数的设置能力。
3. 提供保存与读取模型设置能力。
4. 设置保存需提供成功与失败反馈。

### 4.4 场内策略页面
#### 4.4.1 策略列表与展示
1. 展示场内策略列表，支持桌面表格与移动卡片双形态。
2. 展示策略关键字段，包括标的、策略类型、交易账户、状态、收益等。
3. 提供策略详情展开查看能力。
4. 提供按时间与状态识别的可读化展示。

#### 4.4.2 策略创建
1. 提供创建新策略入口。
2. 支持多种场内策略类型创建，包括基础策略、网格策略、马丁策略、平衡策略等。
3. 支持策略参数表单录入与前端校验。
4. 支持交易账户选择与关联。
5. 创建成功后自动刷新策略列表。

#### 4.4.3 策略操作
1. 提供启动、停止、更新、删除等生命周期操作入口。
2. 提供策略参数修改能力。
3. 提供操作结果反馈与异常提示。
4. 高风险操作需有确认交互，避免误操作。
5. 删除语义应定义为业务下线操作，可通过停止与清仓等流程实现。

#### 4.4.4 辅助能力
1. 提供标的搜索与自动补全能力。
2. 提供策略相关计算结果展示能力。
3. 提供实时价格或关键行情信息展示能力。

### 4.5 场外策略页面
#### 4.5.1 策略列表与展示
1. 展示场外策略列表，支持桌面与移动双形态。
2. 展示基金类策略关键字段，包括组合、阈值、仓位、收益等。
3. 提供详情展开查看与关键信息可视化。

#### 4.5.2 策略创建与编辑
1. 支持场外策略创建，包括基金平衡与基金定投等类型。
2. 支持基金搜索、基金选择、参数配置与账户绑定。
3. 支持策略参数编辑与保存。
4. 支持策略更新后即时刷新显示。

#### 4.5.3 策略操作
1. 提供策略启动、停止、删除等操作。
2. 提供操作反馈与失败原因提示。
3. 删除语义应定义为业务下线操作，可通过停止与清仓等流程实现。

### 4.6 已完成策略页面
1. 展示已完成策略清单。
2. 展示策略完成原因、收益结果、创建时间、策略类型等信息。
3. 支持收益正负样式区分，提升可读性。
4. 支持时间排序与移动端卡片化阅读。
5. 页面配置缺失时需使用默认展示方案。

### 4.7 智能分析页面
#### 4.7.1 对话基础能力
1. 提供用户与智能助手的多轮对话能力。
2. 展示消息时间、消息类型与发送状态。
3. 支持加载中状态与过程提示。
4. 支持 Markdown 内容渲染展示。

#### 4.7.2 上下文增强能力
1. 支持按场景选择上下文类型，包括场内策略、场外策略、其他资产。
2. 支持从用户策略中选择上下文对象。
3. 支持通过资产输入构建资产上下文。
4. 发送问题时自动携带上下文信息，提高回答针对性。

#### 4.7.3 模型能力与参数
1. 提供模型列表加载与模型选择能力。
2. 提供模型密钥设置入口，未配置时引导用户先配置。
3. 支持读取历史模型设置并自动恢复。
4. 支持模型设置保存与即时生效反馈。

### 4.8 券商登录辅助页面
1. 提供交易账户登录表单。
2. 提供验证码获取与验证码输入流程。
3. 提供自动识别验证码能力，减少手工输入。
4. 支持记忆常用账户并快速选择。
5. 登录成功后可返回触发登录前的业务页面，保持流程连续。

## 5. 前端服务功能需求
### 5.1 配置服务
1. 统一维护全局页面配置与分页面配置。
2. 提供配置读取、缓存、失效与兜底能力。
3. 对外提供稳定配置结构，避免页面端适配成本波动。
4. 统一维护页面展示文案字典，保障文案集中管理与一致发布。

### 5.2 文档服务
1. 统一维护文档目录与文档正文内容。
2. 提供文档列表与文档内容查询能力。
3. 支持文档版本更新后的平滑发布，不影响页面端结构。

### 5.3 前端业务聚合服务
1. 统一封装页面调用的业务接口与会话参数处理。
2. 统一处理请求异常、消息提示映射和返回结构标准化。
3. 统一管理配置与文档的本地缓存策略。

## 6. 状态管理与交互规范
### 6.1 会话状态
1. 统一管理访问凭证、用户会话标识、交易会话标识、用户信息。
2. 会话状态变化后需同步更新页面显示与路由权限。

### 6.2 加载与错误处理
1. 关键请求需具备加载状态，避免重复提交。
2. 接口失败需显示可读错误信息，不允许无提示失败。
3. 关键业务动作需有成功提示，增强操作可感知性。

### 6.3 列表与表单体验
1. 列表页需支持空状态与骨架状态。
2. 表单需提供必填校验、格式校验与即时错误提示。
3. 弹窗操作需支持取消与确认的明确行为反馈。

## 7. 非功能需求
### 7.1 性能要求
1. 主要页面首屏加载需控制在可接受范围内。
2. 路由采用按需加载，降低初始包体积。
3. 配置与低频数据应使用缓存减少重复请求。

### 7.2 可用性要求
1. 页面文案、按钮、提示语需一致且易理解。
2. 移动端操作控件尺寸需满足触控场景。
3. 主要流程需在少步骤内完成，减少用户学习成本。

### 7.3 可访问性要求
1. 关键导航区域需具备清晰语义结构。
2. 颜色对比、交互焦点与反馈提示需满足基础可访问要求。

### 7.4 可维护性要求
1. 页面能力按业务域拆分，避免逻辑耦合。
2. 前端服务层职责稳定，页面层按消费契约解耦开发。
3. 公共能力组件化复用，避免重复开发。
4. 支持后续新增页面与策略类型扩展。

## 8. 验收标准
1. 本文档列出的全部页面能力可实现并可测试验证。
2. 前端服务层可独立维护页面配置与文档内容，并向页面层稳定供给。
3. 登录、策略管理、交易账户管理、智能问答、文档阅读等核心链路均可端到端运行。
4. 移动端与桌面端均可完成主要业务操作。
5. 会话校验、权限控制、错误提示在关键流程中全部生效。
6. 配置驱动、主题切换、响应式布局等全局能力可稳定工作。
7. 抽样页面检查不得存在写死业务文案，页面文本均可通过前端服务层配置变更生效。
