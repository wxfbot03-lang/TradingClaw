# TradingClaw 命令行功能需求文档

## 1. 文档目的
1. 定义命令行工具必须具备的完整能力，供团队独立实现与交付。
2. 明确命令结构、使用流程、输出规则、会话管理和验收标准。
3. 作为运维、自动化脚本和终端用户的统一使用规范基线。

## 2. 产品定位与使用场景
### 2.1 产品定位
1. 提供统一终端入口，完成行情查询、交易操作、策略管理、资讯获取等核心业务。
2. 支持人工交互使用，也支持脚本化、批处理和自动化任务集成。
3. 通过本地配置与会话缓存降低重复输入成本，提高操作效率。

### 2.2 典型使用场景
1. 研发和测试人员在本地快速验证后端业务能力。
2. 运维人员在服务器终端执行查询、排障和批量操作。
3. 自动化脚本定时获取行情、资讯、持仓和策略状态。
4. 交易用户在无图形界面环境下完成核心业务流程。

## 3. 全局能力要求
### 3.1 统一命令入口
1. 提供统一命令入口与子命令分组体系。
2. 支持按业务域拆分命令组，包括用户、配置、交易所、资产、订单、策略、资讯。
3. 支持命令帮助信息与示例输出，降低学习成本。
4. 当命令参数错误时，提供可执行示例引导用户纠正输入。

### 3.2 全局参数与执行上下文
1. 支持服务地址、输出格式、格式化输出、会话参数、分页控制、数量限制等全局参数。
2. 支持命令级参数覆盖全局参数。
3. 支持将全局选项透传到各子命令统一生效。
4. 支持全局数量限制用于列表输出裁剪。

### 3.3 输出规范
1. 支持 JSON 与表格两种输出格式。
2. 统一输出状态、消息和业务数据，保证脚本可解析性。
3. 支持在数据量较大时分页输出，避免终端刷屏。
4. 支持关闭分页，以满足管道处理或日志采集场景。
5. 当状态非成功时，命令必须返回非零退出码，便于自动化流程识别失败。

### 3.4 本地配置与会话缓存
1. 提供本地配置读写能力，支持服务地址、令牌、用户会话、交易所会话、交易所类型等关键项。
2. 本地配置目录需自动创建，确保首次使用可落地。
3. 登录成功后自动缓存访问凭证与会话标识，支持后续命令免重复输入。
4. 支持查询当前本地配置与当前默认交易所会话状态。

## 4. 功能需求明细

### 4.1 用户账号能力
#### 4.1.1 登录能力
1. 提供用户名密码登录流程。
2. 登录成功后自动保存访问令牌和用户会话标识。
3. 登录输出需同时覆盖可读信息与脚本可用字段。
4. 登录失败需返回明确错误原因。

#### 4.1.2 资料与会话能力
1. 提供本地用户会话资料读取能力。
2. 当本地会话不存在或已失效时，需提示用户先登录。
3. 提供登出能力，支持显式会话参数与默认会话参数两种模式。

### 4.2 配置管理能力
1. 提供配置项批量或单项写入能力。
2. 提供配置项单项查询与全量查询能力。
3. 支持多种输出格式查看配置内容。
4. 对非法配置键需返回明确错误。
5. 配置写入后应立即对后续命令生效。

### 4.3 交易所会话管理能力
1. 提供用户维度的交易所会话列表查询。
2. 支持展示不同交易所账户与对应会话标识。
3. 提供默认交易所会话查询能力。
4. 提供默认交易所会话设置能力。
5. 提供交易所会话删除能力。
6. 交易所登录与授权流程由图形界面或外部流程完成，命令行侧重点为会话消费与管理。

### 4.4 资产与行情查询能力
#### 4.4.1 资产搜索
1. 提供关键词搜索能力，支持返回多个候选资产。
2. 支持搜索结果数量控制。
3. 支持结果表格化展示，便于人工筛选。

#### 4.4.2 股票行情
1. 提供股票实时价格查询。
2. 提供股票五档盘口查询，并展示买卖方向、价格、数量。
3. 提供股票交易日判断能力。
4. 提供股票历史行情查询，支持周期、起止时间、数量限制。
5. 提供股票均线查询，支持多均线参数与多数据点输出。

#### 4.4.3 基金行情
1. 提供基金实时净值查询。
2. 提供基金历史净值查询。
3. 提供基金均线查询。
4. 支持基金历史查询的默认结束时间推导。

#### 4.4.4 数字资产行情
1. 提供交易对实时价格查询。
2. 提供交易对历史行情查询。
3. 提供交易对均线查询。
4. 支持多别名参数输入，降低交易类命令输入门槛。

#### 4.4.5 持仓查询
1. 提供证券持仓查询，展示资金概览与明细字段。
2. 提供基金持仓查询，支持统一格式输出。
3. 支持输出数量限制与分页展示。

### 4.5 订单与交易能力
#### 4.5.1 证券交易
1. 提供股票限价下单能力。
2. 提供股票市价下单能力。
3. 市价下单需支持交易类型映射与保护价档位校验。
4. 市价下单需支持自动读取盘口并计算实际委托价格。
5. 提供基金下单能力，支持申购/赎回等方向。

#### 4.5.2 证券订单管理
1. 提供单笔撤单能力。
2. 提供全量挂单批量撤单能力。
3. 批量撤单需支持按日期分组执行并汇总结果。
4. 提供挂单、当日委托、当日成交、历史流水查询能力。
5. 提供基金成交历史查询能力。

#### 4.5.3 数字资产订单
1. 提供订单创建能力。
2. 提供订单修改能力。
3. 提供订单状态查询能力。
4. 提供订单撤销能力，支持订单号或客户订单号模式。
5. 提供挂单列表查询与成交明细查询能力。

### 4.6 策略管理能力
#### 4.6.1 策略生命周期
1. 提供策略列表查询能力。
2. 提供策略详情查询能力，支持单策略与多策略聚合查看。
3. 提供策略停止能力，支持附加控制参数。
4. 提供策略重启能力。
5. 提供策略状态更新能力。
6. 提供策略修改能力。

#### 4.6.2 策略创建能力
1. 提供股票网格策略创建。
2. 提供股票简单策略创建。
3. 提供股票马丁策略创建。
4. 提供股票平衡策略创建。
5. 提供基金平衡策略创建。
6. 提供基金定投策略创建。
7. 支持 JSON 字符串和文件两种复杂参数输入方式。

#### 4.6.3 策略输入处理
1. 支持策略类型命名映射，兼容不同命名风格输入。
2. 支持按参数是否传入动态构建请求载荷。
3. 对 JSON 参数解析失败需给出明确错误并阻断执行。
4. 对关键策略操作支持令牌透传，满足安全校验链路。

### 4.7 资讯查询能力
1. 提供多资讯源新闻快讯查询。
2. 支持资讯数量控制与起始游标能力。
3. 支持表格化展示标题、时间和关联标的信息。
4. 支持 JSON 输出用于自动化处理和落库。

### 4.8 命令辅助与易用性能力
1. 提供命令说明、必填参数提示和示例命令输出。
2. 支持分组帮助与子命令帮助。
3. 对常见错误输入提供纠错方向。
4. 支持别名参数，兼容不同用户输入习惯。

### 4.9 构建与分发能力
1. 提供可安装的命令行分发包，安装后可直接执行统一命令入口。
2. 提供标准 Python 打包能力，支持本地构建发布。
3. 提供容器化打包能力，支持在隔离环境生成分发产物。
4. 支持在持续集成或脚本环境中自动产出安装包。

## 5. 非功能需求
### 5.1 稳定性要求
1. 命令执行过程需具备异常捕获与稳定错误返回。
2. 当后端返回异常时，命令不得崩溃退出到无信息状态。
3. 网络异常、参数异常、认证异常需分别给出可定位反馈。

### 5.2 安全要求
1. 令牌与会话信息应使用本地最小化存储，不在普通输出中泄露敏感内容。
2. 关键写操作需具备可扩展的确认机制能力。
3. 命令执行日志和错误提示不得输出明文敏感密钥。

### 5.3 可维护性要求
1. 命令组按业务域拆分，模块边界清晰。
2. 公共能力（请求、输出、参数处理、配置读写）复用统一组件。
3. 新增命令组与新交易通道时不影响既有命令行为。

### 5.4 自动化友好要求
1. 必须支持 JSON 稳定输出，便于脚本直接解析。
2. 必须支持非零退出码表示失败，便于流水线判断。
3. 必须支持参数化会话与环境切换，便于多环境批处理。

## 6. 验收标准
1. 本文档列出的全部命令行能力可实现并可测试验证。
2. 用户、配置、交易所、资产、订单、策略、资讯七大能力域均可稳定运行。
3. JSON 与表格两种输出模式均符合预期。
4. 本地配置缓存、会话透传、错误退出码在关键流程中全部生效。
5. 安装包构建与容器化构建流程可产出可安装命令行工具。
