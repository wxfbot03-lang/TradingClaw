# TradingClaw 命令行需求文档

## 1. 文档说明

### 1.1 文档目标
- 明确 TradingClaw 命令行产品能力范围、业务规则与验收标准。
- 为产品、研发、测试、运维和自动化脚本使用方提供统一需求基线。
- 作为迭代排期、测试设计和交付验收的依据。

### 1.2 适用范围
- 本文档覆盖命令行能力需求、输出规范、会话与配置管理、业务域功能与验收要求。
- 本文档不包含技术架构实现、命令内部实现细节、接口协议实现、数据库设计、打包脚本实现等设计阶段内容。

## 2. 产品目标

### 2.1 产品目标
- 提供统一终端入口，覆盖行情查询、交易操作、策略管理、资讯获取等核心业务。
- 支持人工交互与自动化脚本两类使用模式，保障一致能力输出。
- 通过本地配置和会话缓存降低重复输入，提高终端操作效率。

## 3. 业务范围与总体要求

### 3.1 业务范围
- 用户账号与会话管理
- 本地配置管理
- 交易所会话管理
- 资产与行情查询
- 订单与交易操作
- 策略管理
- 资讯查询
- 命令帮助与可用性能力
- 构建与分发能力

### 3.2 总体业务要求
- 必须提供统一命令入口与按业务域分组的子命令体系。
- 必须支持 JSON 与表格两类输出模式，满足人读与机读双场景。
- 必须支持全局参数与命令级参数覆盖，并在各子命令中统一生效。
- 命令执行失败必须返回非零退出码，确保自动化流程可判定。
- 关键会话信息需本地最小化存储，普通输出不得泄露敏感信息。

### 3.3 版本范围与优先级
- 版本采用 MVP、V1、V2 分层管理。
- 优先级采用 P0、P1、P2，其中 P0 为上线阻断项。
- MVP 冻结后，新增能力默认进入 V1 或 V2。

| 术语 | 定义 | 统一口径 |

## 4. 详细功能需求

### 4.1 统一入口与全局参数（CLI-001）
- 提供统一命令入口与按业务域拆分的子命令分组。
- 支持服务地址、输出格式、格式化开关、会话参数、分页控制、数量限制等全局参数。
- 支持命令级参数覆盖全局参数并按就近原则生效。
- 参数错误时提供可执行示例引导用户纠正输入。
- 支持分组帮助与子命令帮助输出。

### 4.2 输出规范（CLI-002）
- 支持 JSON 与表格两种输出格式。
- 输出结构必须包含状态、消息和业务数据。
- 支持分页输出与关闭分页，适配终端浏览和管道处理场景。
- 支持全局数量限制用于列表结果裁剪。
- 状态非成功时必须返回非零退出码。
- JSON 输出字段约束必须满足第 5.5 节输出契约要求。

### 4.3 本地配置与会话缓存（CLI-003）
- 提供配置项单项与批量写入能力。
- 提供单项查询与全量查询能力。
- 首次使用时自动创建本地配置目录并保证可写。
- 登录成功后自动缓存访问令牌、用户会话标识、交易所会话信息。
- 支持查询当前本地配置与默认交易所会话状态。
- 非法配置键必须返回明确错误信息。

### 4.4 用户账号能力（CLI-004）

#### 4.4.1 登录
- 提供用户名密码登录流程。
- 登录成功后保存访问令牌与用户会话标识。
- 登录输出需同时满足可读展示与脚本字段消费。
- 登录失败需返回明确错误原因。

#### 4.4.2 会话资料与登出
- 提供本地用户会话资料读取能力。
- 本地会话缺失或失效时提示用户先登录。
- 提供登出能力，支持显式会话参数和默认会话两种模式。

#### 4.4.3 用户资料与额度积分查询
- 提供用户资料查询能力，覆盖用户等级、账户摘要等基础信息。
- 提供策略额度查询能力，输出可用额度与已占用额度。
- 提供积分查询能力，输出总积分、已占用积分、可用积分。
- 相关查询结果支持 JSON 与表格双格式输出，字段结构遵循统一输出契约。

### 4.5 交易所会话管理（CLI-005）
- 提供用户维度交易所会话列表查询能力。
- 支持展示交易所账户与对应会话标识。
- 提供默认交易所会话查询与设置能力。
- 提供交易所会话删除能力。
- 删除后应同步更新默认会话状态。

### 4.6 资产与行情查询（CLI-006、CLI-011）

#### 4.6.1 资产搜索
- 提供关键词搜索能力，返回候选资产列表。
- 支持结果数量控制。
- 支持表格化展示便于人工筛选。

#### 4.6.2 股票行情
- 提供股票实时价格查询。
- 提供股票五档盘口查询，展示方向、价格、数量。
- 提供股票交易日判断能力，基于行情服务结果推导，不单独承诺独立后端能力。
- 提供股票历史行情查询，支持周期、起止时间、数量限制。

#### 4.6.3 基金行情
- 提供基金实时净值查询。
- 提供基金历史净值查询。
- 支持历史查询默认结束时间推导。

#### 4.6.4 行情扩展能力（CLI-011，V1）
- 提供股票均线查询，支持多均线参数与多数据点输出。
- 提供基金均线查询。
- 提供交易对实时价格、历史行情、均线查询能力。
- 支持数字资产交易对多别名参数输入。
- MVP阶段执行该类命令时返回能力未开放提示。

#### 4.6.5 持仓查询
- 提供证券持仓查询并展示资金概览与持仓明细。
- 提供基金持仓查询并输出统一字段结构。
- 支持数量限制与分页展示。

### 4.7 订单与交易能力（CLI-007、CLI-012、CLI-013）

#### 4.7.1 证券交易
- 提供股票限价下单能力。
- 提供股票市价下单能力。
- 市价下单支持交易类型映射与保护价档位前置校验。
- 市价下单支持读取盘口并计算委托参考价格。
- 提供基金下单能力，支持申购与赎回方向。
- 提供股票单笔撤单能力。
- CLI负责参数完整性、枚举合法性与保护价范围前置校验。
- 后端负责账户资产、交易时段、风控规则与最终业务校验。

#### 4.7.2 证券订单扩展能力（CLI-013，V1）
- 提供全量挂单批量撤单能力。
- 批量撤单支持按日期分组执行并汇总结果。
- 提供挂单、当日委托、当日成交、历史流水查询能力。
- 提供基金成交历史查询能力。
- MVP阶段执行该类命令时返回能力未开放提示。

#### 4.7.3 数字资产订单（CLI-012，V1）
- 提供订单创建能力。
- 提供订单修改能力。
- 提供订单状态查询能力。
- 提供订单撤销能力，支持订单号或客户订单号模式。
- 提供挂单列表与成交明细查询能力。
- MVP阶段执行该类命令时返回能力未开放提示。

### 4.8 策略管理能力（CLI-008）

#### 4.8.1 策略生命周期
- 提供策略列表查询能力。
- 提供策略详情查询能力，支持单策略与多策略聚合查看。
- 提供策略停止能力，支持附加控制参数。
- 提供策略重启、状态更新、策略修改能力。
- 多策略聚合查看为命令行客户端聚合展示，不依赖后端批量详情接口。
- 多策略聚合查看不作为后端批量详情接口交付承诺。

#### 4.8.2 策略创建
- MVP 仅提供股票基础策略创建能力。
- V1 提供股票网格、股票马丁、股票平衡策略创建能力。
- V1 提供基金平衡与基金定投策略创建能力。
- 支持复杂参数通过 JSON 字符串与 JSON 文件两种方式输入。

#### 4.8.3 策略输入处理
- 支持策略类型命名映射，兼容不同输入风格。
- 支持按参数是否传入动态构建请求载荷。
- JSON 参数解析失败时返回明确错误并阻断执行。
- 关键策略操作支持令牌透传。

### 4.9 资讯查询能力（CLI-009）
- 提供多资讯源新闻快讯查询。
- 支持资讯数量控制与起始游标能力。
- 支持表格展示标题、时间和关联标的信息。
- 支持 JSON 输出用于自动化处理。

### 4.10 易用性与分发能力（CLI-010）
- 提供命令说明、必填参数提示和示例命令输出。
- 对常见错误输入提供纠错方向。
- 支持别名参数，兼容不同输入习惯。
- 高风险写操作需支持交互确认与显式确认参数两种执行方式。
- 在非交互环境中，高风险写操作必须通过显式确认参数执行。
- 提供可安装命令行分发包，安装后可直接执行统一命令入口。
- 提供 Python 标准打包能力与容器化打包能力。
- 支持在持续集成或脚本环境自动产出安装包。

## 5. 非功能需求

### 5.1 稳定性
- 命令执行过程必须具备异常捕获与稳定错误返回，非预期崩溃率每月不高于 0.1%。
- 核心查询命令在正常网络下响应时间 P95 不高于 2.0 秒。
- 写操作命令在正常网络下返回首个可读结果时间 P95 不高于 3.0 秒。
- 后端异常时命令不得无信息崩溃退出，错误可读提示覆盖率不低于 99%。
- 网络异常、参数异常、认证异常必须返回可定位反馈，错误分类准确率不低于 98%。

### 5.2 安全性
- 令牌与会话信息采用本地最小化存储，并遵循第 5.6 节安全口径。
- 普通输出、日志与错误提示不得泄露明文敏感信息，敏感字段脱敏覆盖率为 100%。
- 关键写操作需具备可扩展确认机制能力，并遵循第 5.7 节非交互规范。

### 5.3 可维护性
- 命令组按业务域拆分，模块边界清晰，跨域改动占比控制在 20% 以内。
- 请求、输出、参数处理、配置读写等公共能力统一复用，公共能力复用率不低于 80%。
- 新增命令组与交易通道不影响既有命令行为，兼容性回归通过率需达到 100%。

### 5.4 自动化友好性
- 必须支持稳定 JSON 输出，关键命令 JSON 契约兼容性通过率不低于 99.9%。
- 必须支持非零退出码表示失败，失败命令退出码正确率为 100%。
- 必须支持参数化会话与环境切换，批处理脚本连续执行成功率不低于 99%。
- 分页模式下连续翻页不得出现重复或漏项，稳定性抽检通过率不低于 99.5%。

### 5.5 输出契约约束表
| 字段名 | 类型 | 必填 | 约束说明 |
| --- | --- | --- | --- |
| status | string | 是 | 仅允许 `success` 或 `error` |
| message | string | 是 | 可读结果描述，失败时必须给出明确原因 |
| data | object/array/null | 是 | 业务数据载荷，失败时允许为 null |
| error_code | string/null | 是 | 成功时为 null，失败时必须为可检索错误码 |
| request_id | string | 是 | 单次命令请求追踪标识，需全链路唯一 |
| timestamp | string | 是 | ISO 8601 时间戳，表示命令结果生成时间 |

### 5.6 CLI 安全口径表
| 项目 | 要求口径 |
| --- | --- |
| 本地存储路径 | 配置与会话文件仅允许写入用户目录下专用隐藏目录 |
| 文件权限 | 仅当前用户可读写，禁止其他用户读取会话与令牌文件 |
| 令牌生命周期 | 读取到过期令牌时必须阻断业务并提示重新登录 |
| 令牌清理策略 | 登出后必须清理本地令牌与会话缓存，不得保留可复用凭证 |
| 脱敏规则 | 日志、错误与标准输出中令牌、密钥、密码字段必须全量脱敏 |
| 日志禁打字段 | access_token、refresh_token、api_key、password、secret 及其变体 |

### 5.7 非交互模式规范
- 高风险写操作默认需要确认，交互环境下支持确认提示后执行。
- 非交互环境下禁止触发交互确认，必须通过显式确认参数直接执行。
- 当检测到非交互环境且未提供显式确认参数时，命令必须阻断并返回非零退出码。
- 非交互模式失败提示中必须包含缺失参数名称与可执行示例。
- 显式确认参数仅用于客户端防误操作，不替代后端鉴权、风控与业务校验。
