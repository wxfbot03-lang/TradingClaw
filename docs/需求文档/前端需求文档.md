# TradingClaw 前端需求文档

## 1. 文档说明

### 1.1 文档目标
- 明确 TradingClaw 前端产品能力范围、业务规则与验收标准。
- 为产品、研发、测试提供统一需求基线，降低实现偏差。
- 作为迭代计划、测试用例设计与验收评审的依据。

### 1.2 适用范围
- 本文档覆盖前端页面能力、前端服务能力、状态管理与通用交互要求。
- 本文档不包含视觉设计稿、技术架构实现、组件实现细节、接口协议设计等设计阶段内容。

## 2. 产品目标

### 2.1 产品目标
- 提供覆盖认证、策略管理、账户管理、文档阅读、智能分析的一站式前端工作台。
- 支持桌面端与移动端主要业务可用，保证核心链路可连续完成。
- 通过配置驱动与统一前端服务，提升页面可维护性与迭代效率。

## 3. 业务范围与总体要求

### 3.1 业务范围
- 应用壳与全局导航
- 会话与权限控制
- 主题与响应式能力
- 配置驱动页面能力
- 首页与文档中心
- 用户认证
- 账户设置
- 场内策略管理
- 场外策略管理
- 已完成策略查看
- 智能分析
- 券商登录辅助
- 前端配置服务、文档服务、业务聚合服务

### 3.2 总体要求
- 页面文案必须通过前端服务层供给，页面层不得写死业务文案。
- 页面层必须通过前端服务层获取配置与文档数据，不得直接维护源数据。
- 关键操作必须具备加载状态、成功反馈与失败反馈。
- 会话失效时，所有依赖会话的页面必须引导用户重新登录。

### 3.3 需求分层与优先级
- 版本分层：MVP、V1、V2。
- 优先级分层：P0（上线阻断项）、P1（核心增强项）、P2（优化扩展项）。

### 3.5 业务边界与排除项
| 分类 | 本期包含 | 本期不包含 | 排除原因 |
| --- | --- | --- | --- |
| 前端页面能力 | 认证、策略、账户、文档、智能分析、券商辅助 | 可视化大屏、营销活动运营后台 | 不属于当前交易工作台核心范围 |
| 前端服务层 | 配置、文档、业务聚合、缓存策略 | 多租户配置中心、跨系统配置编排 | 复杂度高，需独立规划 |
| 智能分析 | 对话、上下文增强、模型设置 | 自动投资决策托管、收益承诺能力 | 合规与风险边界限制 |
| 交易辅助 | 账户登录、验证码处理、回跳流程 | 交易指令代执行、自动风控处置 | 与交易后端能力边界不一致 |

### 3.6 核心术语表
| 术语 | 定义 | 统一口径 |
| --- | --- | --- |
| 业务下线删除 | 将策略从可操作状态移出，不再作为活跃策略管理 | 删除不等于物理清除，按下线语义处理 |
| 会话恢复 | 用户刷新页面后恢复登录身份并继续操作 | 必须恢复访问凭证、用户信息与路由权限状态 |
| 交易账户 | 用户可用于执行交易的账户实体 | 必须先完成账户归属校验后才能执行交易操作 |
| 交易所会话 | 用户在当前登录态下可用的交易调用上下文 | 失效后必须引导重新建立有效会话 |
| 兜底文案 | 配置读取失败或缺失时展示的默认业务文案 | 统一由前端服务层维护并发布 |
| 配置驱动 | 页面文本与展示规则由配置供给 | 页面层仅消费配置，不维护源文案 |
| 核心链路 | 认证、账户、策略、智能分析、文档阅读主流程 | 核心链路失败必须反馈可读错误原因 |

### 3.7 前端文档中心供给边界说明
- MVP 阶段文档中心由前端静态配置或静态资源承接，不依赖后端文档服务接口。
- 前端文档中心仅承诺静态展示、目录切换与阅读能力。
- 如需后端动态供给文档内容，须新增独立需求ID并纳入版本计划。

## 4. 详细功能需求

### 4.1 应用壳与全局能力（FRT-001~FRT-004）

#### 4.1.1 应用壳与导航（FRT-001）
- 提供统一头部、导航区与主内容区的页面承载能力。
- 提供桌面端与移动端可用的导航进入方式。
- 提供核心业务入口，保证页面跳转逻辑一致。
- 提供全局退出登录能力，退出后清理会话并跳转认证页。

#### 4.1.2 会话与权限（FRT-002）
- 支持登录态持久化，保存访问凭证、会话标识、用户信息。
- 支持路由访问控制，未登录用户仅可访问公开页面。
- 支持页面刷新后的登录身份恢复。
- 会话失效时，依赖会话页面必须阻断并引导重新登录。

#### 4.1.3 主题与响应式（FRT-003）
- 支持深色主题与浅色主题切换。
- 支持主题跟随系统设置自动切换。
- 主要业务页面支持移动端自适应可用。
- 跨页面视觉风格需保持一致性与可读性。

#### 4.1.4 配置驱动能力（FRT-004）
- 支持全局配置与页面级配置读取。
- 支持配置缓存，减少重复读取并提升加载效率。
- 配置读取失败时，必须使用默认兜底文案保证页面可用。
- 默认兜底文案由前端服务层统一维护。

### 4.2 首页与文档中心（FRT-005）
- 展示首页核心介绍信息、关键入口与品牌信息。
- 支持文档目录列表展示与目录切换阅读。
- 首次进入文档中心时默认打开首个文档。
- 支持文档正文的结构化内容展示。
- 提供文档加载中、加载失败、空数据三类状态提示。
- 移动端支持目录与阅读区切换。

### 4.3 用户认证（FRT-006）

#### 4.3.1 注册
- 提供注册表单并执行前端输入校验。
- 注册结果需明确反馈，成功后引导进入登录流程。

#### 4.3.2 登录
- 提供登录表单并执行字段合法性校验。
- 登录成功后保存访问凭证与用户会话信息。
- 登录成功后跳转默认业务页。
- 登录失败时展示明确失败原因。

### 4.4 账户设置（FRT-007、FRT-014）

#### 4.4.1 交易账户管理
- 展示当前用户已绑定交易账户列表。
- 提供新增交易账户入口，并支持完成账户接入流程。
- 提供删除交易账户能力。
- 账户变更后自动刷新列表状态。

#### 4.4.2 模型设置管理
- 展示当前模型配置状态。
- MVP 阶段支持模型参数展示与配置引导，不提供写入能力。
- V1 阶段支持模型密钥与自动登录参数写入。
- V1 阶段支持模型设置读取、保存与生效反馈。

### 4.5 场内策略（FRT-008、FRT-015）

#### 4.5.1 策略列表与展示
- 支持场内策略列表展示，覆盖桌面与移动端可读形态。
- 展示关键字段：标的、策略类型、交易账户、状态、收益。
- 支持策略详情展开查看。
- 支持时间与状态信息的可读化展示。

#### 4.5.2 策略创建
- 提供创建新策略入口。
- MVP 阶段仅支持基础策略创建。
- 网格策略、马丁策略、平衡策略等类型扩展在 V1 阶段交付。
- 支持参数录入与前端校验。
- 支持交易账户选择与关联。
- 创建成功后自动刷新策略列表。

#### 4.5.3 策略操作
- 支持启动、停止、更新、删除等生命周期操作。
- 支持策略参数修改。
- 操作后需返回明确成功或失败反馈。
- 高风险操作必须要求确认，防止误操作。
- 删除语义定义为业务下线操作。

#### 4.5.4 辅助能力
- 支持标的搜索与自动补全。
- 支持策略相关计算结果展示。
- 支持实时价格或关键行情信息展示。

### 4.6 场外策略（FRT-009）

#### 4.6.1 策略列表与展示
- 展示场外策略列表，支持桌面与移动端可用。
- 展示组合、阈值、仓位、收益等关键字段。
- 支持详情展开与关键信息可读化展示。

#### 4.6.2 策略创建与编辑
- 支持基金平衡与基金定投策略创建。
- 支持基金搜索、基金选择、参数配置与账户绑定。
- 支持策略参数编辑与保存。
- 更新后需即时刷新显示结果。

#### 4.6.3 策略操作
- 支持策略启动、停止、删除。
- 操作后需反馈结果与失败原因。
- 删除语义定义为业务下线操作。
- 后端能力未开放时页面需展示能力未开放状态，不得以异常弹窗替代。

### 4.7 已完成策略（FRT-010）
- 展示已完成策略清单。
- 展示完成原因、收益结果、创建时间、策略类型等信息。
- 支持收益正负差异化表达，提升可读性。
- 支持时间排序与移动端阅读友好展示。
- 页面配置缺失时需启用默认展示方案。
- 后端能力未开放时页面需展示能力未开放状态并保留返回路径。

### 4.8 智能分析（FRT-011）

#### 4.8.1 对话基础能力
- 支持用户与智能助手多轮对话。
- 展示消息时间、消息类型与发送状态。
- 支持消息加载中状态与过程提示。
- 支持 Markdown 内容展示。

#### 4.8.2 上下文增强能力
- 支持按场景选择上下文类型：场内策略、场外策略、其他资产。
- 支持从用户策略中选择上下文对象。
- 支持通过资产输入构建上下文。
- 发送问题时自动携带上下文信息。

#### 4.8.3 模型能力与参数
- 支持模型列表加载与模型选择。
- 提供模型密钥设置入口，未配置时给出引导。
- 支持读取历史模型设置并自动恢复。
- 支持模型设置保存并即时反馈生效状态。
- 后端能力未开放时页面需展示能力未开放状态并保留历史对话只读查看。

### 4.9 券商登录辅助（FRT-012）
- 提供交易账户登录表单。
- 提供验证码获取与输入流程。
- 支持验证码自动识别能力。
- 支持记忆常用账户并快速选择。
- 登录成功后可返回触发登录前的业务页面。
- OCR 或辅助登录能力未开放时，页面需展示能力未开放并保留外部登录引导。

### 4.10 前端服务层（FRT-013）

#### 4.10.1 配置服务
- 统一维护全局配置与页面配置。
- 支持配置读取、缓存、失效、兜底能力。
- 对外输出稳定配置结构，降低页面适配波动。
- 统一维护页面展示文案字典。

#### 4.10.2 文档服务
- 统一维护文档目录与文档正文内容。
- 支持文档列表与文档内容查询。
- 支持文档更新后的平滑发布。
- MVP 阶段文档内容由前端静态资源承接，不依赖后端动态文档接口。

#### 4.10.3 前端业务聚合服务
- 统一封装页面调用的业务接口与会话参数处理。
- 统一处理请求异常、消息提示映射、返回结构标准化。
- 统一管理配置与文档的本地缓存策略。

## 5. 状态管理与交互规范

### 5.1 会话状态
- 统一管理访问凭证、用户会话标识、交易所会话标识、用户信息。
- 会话状态变化后，页面显示与路由权限需同步更新。

### 5.2 加载与错误处理
- 关键请求必须具备加载状态，防止重复提交。
- 接口失败必须显示可读错误信息。
- 关键业务动作必须有成功提示。

### 5.3 列表与表单体验
- 列表页必须支持空状态与骨架状态。
- 表单必须支持必填校验、格式校验、即时错误提示。
- 弹窗操作必须具备取消与确认两类明确反馈。

## 6. 非功能需求

### 6.1 性能
- 首页与核心业务页首屏加载时间 P95 不高于 2.5 秒。
- 路由切换到目标页面的可交互时间 P95 不高于 1.2 秒。
- 配置与低频数据读取缓存命中率不低于 70%。
- 文档列表与策略列表页面首次数据可见时间 P95 不高于 1.5 秒。

### 6.2 可用性
- 页面文案、按钮、提示语口径一致性抽样通过率不低于 98%。
- 关键操作反馈时延 P95 不高于 500ms。
- 关键失败场景错误提示可读率不低于 99%。
- 移动端核心流程完成率不低于 95%。

### 6.3 可访问性
- 关键导航区域需具备清晰语义结构，抽样页面覆盖率 100%。
- 键盘可聚焦关键交互控件覆盖率不低于 95%。
- 反馈提示需支持文本可感知表达，抽样缺失率为 0。

### 6.4 可维护性
- 页面能力按业务域拆分，跨域耦合改动占比控制在 20% 以内。
- 前端服务层职责稳定，页面层按消费契约解耦开发。
- 公共能力组件化复用率不低于 80%。
- 新增页面与策略类型需求不应修改既有消费契约。

## 7. 验收标准

### 7.1 总体验收要求
- 本文档全部功能需求均可实现并可测试验证。
- 前端服务层可独立维护配置与文档内容，并稳定供给页面层。
- 登录、策略管理、交易账户管理、智能分析、文档阅读等核心链路可端到端运行。
- 桌面端与移动端均可完成主要业务操作。
- 会话校验、权限控制、错误提示在关键流程全部生效。
- 配置驱动、主题切换、响应式布局等全局能力稳定工作。
- 抽样页面检查不允许存在写死业务文案，页面文案应可通过前端服务层配置生效。
